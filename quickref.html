Ways of working
🎨 Model A — Designer vibes, dev integrates
Designer ships UI components via Claude Code. Dev handles API, state, auth. Works when: design system is tokenised in code and there's a clear UI/logic boundary. Breaks when: components don't follow codebase conventions.
⌨️ Model B — Designer designs, dev vibes
Traditional roles, dev uses Claude Code with Figma MCP to implement from spec. Works when: Figma files are well-structured and there's a shared review ritual. Breaks when: Figma is messy and drift goes unnoticed.
🔄 Model C — Fluid, explicit ownership
Both use Claude. The question isn't who writes — it's who owns the PR. Works at high trust. Collapses without code ownership culture.
The failure mode all three share: no one owns the design-code parity check. Build that ritual first.
