Setup

Getting Everything Running

⚡ npm install is deprecated. Use the native installer — no Node.js, auto-updates silently. See official install docs ↗
01
Install Claude Code — code.claude.com/docs ↗
macOS / Linux: curl -fsSL https://claude.ai/install.sh | bash
Windows (PowerShell): native installer — see docs ↗ · requires Git for Windows.
Homebrew: brew install claude-code (run brew upgrade claude-code manually for updates)
Requires Claude Pro ($20/mo) or Max ($100+/mo). No free tier.
02
Add Figma MCP (Remote) — Figma Remote MCP Docs ↗
claude mcp add --transport http figma https://mcp.figma.com/mcp
Add --scope user to make it global across all projects (do this).
Type /mcp in Claude Code → select figma → Authenticate → Allow Access. ✓
03
Or: install the official Figma plugin — plugin docs ↗
claude plugin install figma@claude-plugins-official
Bundles MCP + Agent Skills in one shot. Recommended if starting fresh.
04
Enable Dev Mode in Figma — figma.com/dev-mode ↗
Click </> in Figma toolbar. For local desktop MCP: Figma menu → Preferences → Enable Dev Mode MCP Server. Runs at 127.0.0.1:3845/sse.
⚠ Free plan: ~6 tool calls/month with official MCP. Paid plan for real work.
Setup · Prep Your File

Prep Your Figma File
or Claude will make some interesting decisions

🚨 A file full of Frame 1234 and Rectangle 69 is like asking Claude to read your mind. It'll try. The output will be... creative. And wrong.
🏷️
Name Layers Semantically
Use: Button/Primary/Large, Card/Product. Claude infers component hierarchy from layer names. Figma's bulk rename plugin is your best friend here.
🎨
Use Figma Variables ↗
Variables → export JSON → Claude maps color/brand/primary to your token system. Output uses tokens, not #9747FF hardcoded everywhere.
🧩
Variants = Props
Button with variant=primary/ghost + size=sm/md/lg → clean typed React props. Figma variants map 1:1 to component props. Design it right, get it right.
📐
Auto Layout ↗ = Flexbox
Claude reads Auto Layout as flexbox/grid perfectly. Manual absolute positioning → position:absolute soup. Don't do that to anyone.
🔗
Code Connect ↗
Link Figma components to real code paths. Claude picks up the mapping and references your actual component library — not inventions.
📌
Node URLs, Not Page URLs
Right-click a component → Copy link. Node URL = specific context. Page URL = noise. One component = clean output. Every time.
The Workflow

Figma → Claude → Code
The loop that actually ships things

1
📎 Copy the Figma node link
Right-click a frame/component → Copy link. URL includes the node ID. Paste in Claude Code ↗ or Claude.ai ↗. Add context: "Next.js 15 + Tailwind. Tokens in /tokens/index.ts."
2
🔍 Claude fetches design data via MCP ↗
Via mcp.figma.com/mcp ↗, Claude reads: dimensions, fills, typography, Auto Layout constraints, component variants, Variable values. No manual spec sheet. No phone tag with design.
3
💬 Prompt it properly (next slide has the template)
"Implement this ProductCard as a React component. Props: title, price, imageUrl, badge?. Tailwind. Tokens from /tokens. Hover adds elevation. ARIA-compliant. Add a Storybook story."
Claude is powerful. Not psychic.
4
✅ Review → Iterate → Ship
Claude Code writes to your repo. Review the diff. Tweak in natural language. Design updated? Paste new link — Claude diffs and updates only what changed. git commit. Sprint ticket closed. You take the credit.
🏗️ Selection-based mode: Select a frame in Figma Desktop, prompt Claude Code. It reads your current Figma selection automatically — more precise than pasting links for complex work.
Workflow · Prompting

The Art of Prompting for UI

💬 Claude is a brilliant senior dev who has never seen your product. Context = better output. Prompt like you're onboarding a new hire — in 30 seconds flat.
❌
Weak Prompt
"Make this look like the Figma design."

Claude: *generates a generic card that vaguely gestures in the right direction*

You: *quiet despair*
✅
Strong Prompt
"Implement this ProductCard (Figma link) as React. Props: title, price, imageUrl, badge?. Tailwind CSS. Tokens at /tokens/spacing.ts. Hover → elevation shadow. Keyboard nav + ARIA. Storybook story for all variants."

✦ Power Moves

  • Reference existing code: "Follow the pattern in /components/Card.tsx"
  • Bundle the whole ask: component + unit tests + Storybook in one prompt
  • Specify responsive breakpoints explicitly (sm/md/lg/xl)
  • Screenshot + Figma link together: visual intent + technical data
  • Tell it what NOT to do: "No inline styles. No magic numbers."
  • Give Claude a role: "Act as a senior FE dev who cares about a11y and clean diffs"

Honest Assessment · cont.

What Claude CANNOT Do

✓ Reliably Works
Pixel-perfect from well-named, structured files
Map Figma Variables to code tokens
Understand design intent via semantic naming
Generate ARIA-compliant, keyboard-navigable markup
Responsive layouts from Auto Layout constraints
Iterative updates when design changes
✗ Don't Expect This
Mind-reading unnamed, structureless Figma files
Replacing your design judgment
100% production-ready on literal first pass
Multi-screen flows in one single shot
Knowing your business logic or backend
Exporting Lottie/AE animations from static frames
🤔 The #1 misconception at every demo: "Just give it the link and it builds the app." It's a power tool, not a magic wand. You're still the designer. Claude just never files a Jira ticket asking what "make it feel lighter" means. — Taroon

The Reverse Loop Feb 2026

Code → Figma
Now officially real. Figma blog ↗

🎉 In February 2026, Figma officially launched Claude Code → Figma ↗. Build in code, send to Figma as editable frames. Two-way sync is no longer a dream.
A
🖥️ Capture Live UI → Editable Figma Frame
Run your app locally. Ask Claude Code to capture the current browser state and send it to Figma. Each captured screen becomes a real editable Figma frame — not a flat image. Capture entire flows in one session. More ↗
B
🎨 Design in Figma → Pull Back to Code
Build in Claude Code → polish on the Figma canvas → iterate back via MCP. Teams can annotate, call out what's unclear, explore divergent ideas — without touching the codebase. The canvas is now a thinking space, not just a handoff artifact.
C
🔄 Token Sync — Code → Figma Variables
Export CSS/JS tokens as JSON → Claude converts to Figma Variable format ↗ → push via Figma REST API ↗. Design system stays in sync. No custom plugin project needed.
💡 The emerging loop: Start in code → capture to Figma → team reviews on canvas → refine → back to code. Speed without losing the shared visual thinking layer.
Pro Tips · by Taroon

Things I Learned
The Hard Way

📄
Create a CLAUDE.md ↗ Do This First
File in your repo root. Stack, token paths, conventions, what NOT to do. Claude reads it every session. 30-second onboarding that actually sticks. Commit it. Team-wide instant benefit.
🌐
--scope user for MCP
When adding Figma MCP, use --scope user. Available across all your projects. No reinstall per repo. One-time setup. docs ↗
🧵
One Component Per Convo
Don't ask for 12 components at once. Context dilutes. One component, crisp output. Repeat 12 times. Feels slower. Ships faster. You will trust this eventually.
📸
Screenshot + Link Combo
Paste a screenshot AND the Figma node link together. Screenshot = visual intent. Link = technical data. Two inputs → noticeably better output. Worth the 3 extra seconds.
🎭
Give Claude a Role
"Act as a senior FE dev who deeply cares about performance and a11y." Approach and output quality shift. Engineering leads will notice. In a good way.
🔌
Code Connect ↗ is Worth It
Link Figma components to real code components. Claude knows which component in your repo maps to which Figma node. Output references your actual lib, not invented components.
Agent Skills · Section 1 of 3

Agent Skills —
Claude's Plugin System

MCP gives Claude new tools. Skills give Claude new expertise. They're not the same thing — use both.

📄
What is a Skill?
A directory containing a SKILL.md file — YAML frontmatter + markdown instructions. That's literally it. Claude reads it like an expert onboarding doc.
⚡
Progressive Disclosure
Only the name + description load at startup (~100 tokens). Full instructions load when a task matches. Reference files load only if needed. Zero context tax.
🌐
Cross-Platform
One npx skills add installs to Claude Code, Cursor, Codex, VS Code Copilot, Gemini CLI and 35+ other agents simultaneously. Write once, runs everywhere.
# Install to Claude Code (skills auto-detect your agents)
npx skills add anthropics/skills@frontend-design
# List all installed skills
npx skills list
# Invoke in Claude Code by slash command or it auto-triggers
/frontend-design   or just describe a frontend task
🧠
Skills vs MCP — Know the Diff
MCP = kitchen equipment (new tools: Figma API, web search, database access).
Skills = recipes (how to use them: design review protocol, token workflow, component standards).
Combine them. That's where the real power is.
📁
Where Skills Live
~/.claude/skills/ — personal, all projects
.claude/skills/ — project, shared via git
Skills in .claude/skills/ are committed with your repo — teammates get them on clone. No separate install.

Agent Skills · Section 2 of 3

Building a SKILL.md
It's a markdown file. You've absolutely got this.

📁
Folder Structure
my-skill/
├── SKILL.md         (required)
├── scripts/         Python / Bash
├── references/     docs, loaded on-demand
└── assets/         templates, icons, fonts
🎯
Description is the Trigger
Claude matches tasks to skills using the description field alone. Be specific and a little pushy: "Use this whenever the user mentions X, Y, or Z — even if they don't say the skill name." Undertriggering is the #1 failure mode.

✦ A working SKILL.md — design review for Taroon's product:

---
name: design-review
description: Use this when asked for design review, UI critique, a11y
  check, or visual feedback. Triggers on "review this", "how does
  this look", "accessibility", or any UI feedback request.
---
# Design Review Protocol
Always evaluate: contrast (WCAG AA), touch targets (44×44px min),
spacing scale consistency, reduced-motion-safe transitions.
Output: severity-tagged list 🔴 critical / 🟡 warn / 🟢 ok
Detailed token rules: see references/design-tokens.md
🤖
Let Claude Write It
Run npx skills add anthropics/skills --skill skill-creator then type /skill-creator. Claude interviews you, drafts SKILL.md, runs 20 eval queries, and iterates until it triggers correctly. Minutes, not hours.
🔁
Test It Properly
Write 20 test prompts — half should trigger, half shouldn't. Ask Claude to evaluate both sets. If precision is below 80%, tighten the description. Don't skip this. Skills that don't trigger are skills that don't exist.
📤
Ship It
Commit to .claude/skills/ for your team. Push to GitHub + list on skills.sh ↗. Others install with npx skills add you/repo. Version it. Bump the version field for auto-updates to work.
💡 The meta move — Taroon's recommendation: encode your design system rules, naming conventions, token paths, and review checklist as a single skill. Every time Claude touches your product — whether it's a teammate or you — it follows your standards automatically. Zero re-prompting. Forever.

Bonus Tool · Community Pick ★ 436

figma-console-mcp
Your design system as an API

🔌 This is NOT the official Figma MCP. It's a community-built server by southleft ↗ that goes much deeper — 56+ tools, live plugin debugging, design creation, variable management, and design-code parity checks. Use it alongside the official one.
🐛
Plugin Debugging
Capture real-time console logs, errors, and stack traces from Figma plugins. Watch logs stream as code runs. Essential for plugin devs.
✏️
Design Creation
Tell Claude to "Create a modal with header, body, and action buttons" — it creates actual Figma frames. Not read-only. It writes.
🔧
Variable Management
Full CRUD on Figma Variables. Create collections, add modes (Light/Dark), batch-create 100 tokens in one call. No Enterprise plan needed.
📊
Design System Dashboard
Lighthouse-style health scorecard — naming, tokens, components, a11y, consistency, coverage. Scored 0–100. Ask: "Audit my design system."
🎨
Token Browser (MCP App)
Interactive UI rendered inline in Claude. Browse tokens by collection, filter by type, switch modes. Click-to-copy values. No context used.
🔍
Design–Code Parity
Compare a Figma component spec against your actual code implementation. Returns a scored diff with actionable fix items. Pre sign-off gold.
