setup/install.sh

Claude Code native binary. No Node.js dependency. Auto-updates in background. Official docs ↗

🚫 npm install is deprecated. npm install -g @anthropic-ai/claude-code still works but won't auto-update and requires Node.js. Use the native installer below.

macOS / Linux

bash
# Native installer — fastest path, no deps curl -fsSL https://claude.ai/install.sh | bash # Verify claude --version

macOS via Homebrew

bash
brew install claude-code # Homebrew does NOT auto-update — run periodically: brew upgrade claude-code

Windows (PowerShell)

powershell
# Requires Git for Windows — install first from git-scm.com # Then run in PowerShell (not CMD): winget install Anthropic.ClaudeCode # Or native installer: irm https://claude.ai/install.ps1 | iex

Authentication

bash
# First run — opens browser for OAuth claude # Or set API key directly (for CI/CD) export ANTHROPIC_API_KEY=sk-ant-...
⚠️ Requires a paid plan. Claude Pro ($20/mo) or Max ($100+/mo). No free tier for Claude Code. API key from console.anthropic.com also works — pay per token.
🩺 Run claude doctor after install to verify connectivity, auth status, and installation type.

setup/mcp-config.md

Connect Claude Code to Figma via the official Remote MCP server. Figma docs ↗

Option A — Official Figma Plugin (recommended)

bash
# Installs MCP + Agent Skills in one shot claude plugin install figma@claude-plugins-official # Verify /mcp

Option B — Manual Remote MCP

bash
# Add for current project only claude mcp add --transport http figma https://mcp.figma.com/mcp # Add globally (recommended — available in all projects) claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp # Authenticate # Type /mcp in Claude → select figma → Authenticate → Allow Access

Option C — Desktop MCP (local, offline-friendly)

bash
# Requires Figma Desktop app # Figma menu → Preferences → Enable Dev Mode MCP Server # Server runs at: http://127.0.0.1:3845/sse # Connect Claude Code to it: claude mcp add --transport sse figma-local http://127.0.0.1:3845/sse

Verify connection

bash
# In Claude Code — shows all connected MCP servers /mcp # Should show: figma ● connected
⚠️ Free plan limit: ~6 MCP tool calls/month with the official server. For real usage, upgrade to Figma Professional. Alternatively use the community figma-console-mcp which has no such cap.

setup/figma-prep.md

Garbage in, garbage out. How you structure your Figma file directly determines code quality. Claude reads layer names, variant properties, Auto Layout constraints, and Variables — not pixels.

Layer naming

figma
# Bad — Claude has to guess Frame 1234 / Rectangle / Group 3 # Good — Claude infers component hierarchy Button/Primary/Large Card/Product/Default Nav/TopBar/Authenticated

Component variants → code props

figma → react
// Figma variant properties: variant=primary | ghost | destructive size=sm | md | lg state=default | hover | disabled // Maps directly to typed React props: interface ButtonProps { variant: 'primary' | 'ghost' | 'destructive' size: 'sm' | 'md' | 'lg' disabled?: boolean }

Auto Layout = flexbox

figma → css
/* Auto Layout (horizontal, gap 8) → */ .component { display: flex; flex-direction: row; gap: 8px; align-items: center; } /* Manual absolute positioning → position:absolute soup */ /* Don't do this in Figma. Seriously. */

Figma Variables → Design Tokens

bash
# Export Variables as JSON from Figma → Plugins → Variables to JSON # Then ask Claude: "Convert this variables JSON to CSS custom properties and a JS token file" # Result: tokens/index.ts, tokens/variables.css # Claude will reference these in all generated components
🔗 Use Code Connect ↗ to link Figma components to your actual codebase components. Claude will reference your real component paths instead of generating new ones from scratch.

workflow/loop.md

The full Figma → Claude → Code loop. Each step is atomic — you can enter at any point.

Step 1: Get the Figma node URL

figma
# Right-click any frame or component → Copy link # Node-specific URL format: https://figma.com/design/FILE_KEY/NAME?node-id=NODE_ID # NODE_ID is the key — it points Claude to the exact component # Whole-page URLs are too noisy. Always use node-specific links.

Step 2: Paste in Claude Code with context

prompt
Implement the ProductCard component from this Figma link: https://figma.com/design/abc123/DS?node-id=4-23 Stack: Next.js 15, Tailwind CSS, TypeScript Tokens: /tokens/index.ts Follow patterns in: /components/Card.tsx Props needed: title, price, imageUrl, badge?: string Add hover elevation matching the hover variant in Figma ARIA-compliant. Include a Storybook story for all variants.

Step 3: Claude reads via MCP

output
# Claude fetches from mcp.figma.com/mcp: # - Node dimensions, fills, border-radius # - Auto Layout constraints → flexbox rules # - Typography styles → font-size, weight, line-height # - Variant properties → TypeScript interface # - Variable bindings → token references # Then writes: components/ProductCard.tsx + ProductCard.stories.tsx

Step 4: Design updated?

prompt
The ProductCard design was updated — new padding and a new "sold out" state was added. Updated Figma link: https://figma.com/design/abc123/DS?node-id=4-23 Diff the current /components/ProductCard.tsx against the new design and apply only the changed properties. Don't regenerate the whole component.
🏗️ Selection-based mode: In Figma Desktop, select a frame, then prompt Claude Code. It reads your current Figma selection automatically — more precise than pasting links for complex nested components.

workflow/prompts.md

Copy-paste prompt templates. Adapt the paths and stack specifics to your project.

Component from Figma

prompt template
Implement [COMPONENT_NAME] from [FIGMA_URL] as a React component. Stack: [YOUR_STACK] Tokens: [TOKEN_FILE_PATH] Pattern reference: [EXISTING_COMPONENT_PATH] Props: [LIST_PROPS] Requirements: - Match all Figma variant states (hover, focus, disabled) - Keyboard navigable, ARIA-labelled - No inline styles, no hardcoded values - Export a Storybook story for each variant - TypeScript strict mode

Token extraction

prompt template
Extract all design tokens from [FIGMA_URL]. Output three files: 1. tokens/variables.css — CSS custom properties 2. tokens/index.ts — typed JS token object 3. tokens/tailwind.ts — Tailwind theme extension Group by: color, spacing, typography, radius, shadow. Use kebab-case for CSS vars, camelCase for JS keys.

Iterative update

prompt template
The design for [COMPONENT] changed. Updated link: [FIGMA_URL] Read the current [FILE_PATH] and apply only the visual changes. Preserve all logic, state management, event handlers, and prop interfaces. Output a minimal diff.

What makes a prompt fail

ProblemFix
Whole-page Figma URLUse node-specific URL (?node-id=X-Y)
No stack contextSpecify framework + CSS approach
No token referencePoint to your actual token file
12 components at onceOne component per conversation
No pattern referenceLink to an existing similar component

workflow/limits.md

Accurate model of what Claude does and doesn't do well. Set expectations before the demo.

Reliable

✓ works well
✓ Component generation from well-structured Figma files ✓ Token extraction → CSS custom properties, JS/TS ✓ Auto Layout → flexbox/grid with correct constraints ✓ Variant properties → TypeScript interfaces ✓ Hover/focus/disabled states from Figma variants ✓ Storybook story generation for all variants ✓ Iterative updates — diff and patch, not full regen ✓ ARIA roles and keyboard navigation ✓ Tailwind, CSS Modules, styled-components — all work

Won't work / needs workarounds

✗ known limits
✗ Unnamed layers (Frame 1234) — Claude reads layer names as component hierarchy. "Frame 1234" gives it nothing to infer from. ✗ Business logic — Claude sees your UI, not your data model, API contracts, or state management. Wire those yourself. ✗ Multi-screen flows in one shot — context fills up fast. One component or one screen per conversation. ✗ Lottie / After Effects animation export — no render pipeline. ✗ 100% production-ready on first pass — treat output as a strong first draft. Always review before merging. ✗ Mind-reading: vague prompts = vague output. Always will be.
ℹ️ The #1 misconception: "paste the Figma link and it builds the app." Claude is a precision tool, not a magic wand. Quality of output is a direct function of quality of Figma structure + quality of prompt. — Taroon Tyagi

workflow/code-to-figma.md Feb 2026

Capture live UI from your running app directly into editable Figma frames. Figma blog post ↗

Capture running app → Figma

bash
# 1. Run your app locally npm run dev # 2. In Claude Code, ask it to capture and send to Figma: "Capture the current state of localhost:3000/dashboard and send it to my Figma file [FIGMA_URL] as a new frame named 'Dashboard - Captured [date]'" # Result: editable Figma frame, not a flat image # Teams can annotate, fork, compare against spec

Token sync: code → Figma Variables

bash
# Export tokens as JSON npx token-transformer tokens/index.ts tokens/output.json # Ask Claude to push to Figma Variables via REST API "Convert tokens/output.json to Figma Variables format and push to file [FILE_KEY] via the Figma REST API. Use FIGMA_TOKEN env var for auth."

workflow

Designers refine in Figma ────────► Updated spec 5. Pull back via MCP ────────────────► Claude Code 6. Iterate ──────────────────────────► Repeat

skills/intro.md

Agent skills are modular instruction sets that extend Claude's behaviour. A skill = a SKILL.md file. Official docs ↗ · skills.sh ↗

How skills load — why this matters for context

loading order
Startup: name + description loaded (~100 tokens each) On match: full SKILL.md body loaded into context On demand: references/ files loaded only if task needs them Scripts: scripts/ files executed via bash — only output consumed Result: install 50 skills — zero context penalty until triggered. Why it matters: Claude has a fixed context window. Skills that load lazily mean you get broad capability without burning tokens on instructions Claude doesn't need for your current task.

Install

bash
# Install a skill (auto-detects Claude Code, Cursor, Codex, etc.) npx skills add anthropics/skills@frontend-design # Install to Claude Code specifically npx skills add vercel-labs/agent-skills -a claude-code # --scope user = available in ALL your projects (recommended) # default (no flag) = current project only (.claude/skills/) # List installed npx skills list # Update all npx skills update # Invoke by slash command /frontend-design

Skill install locations

ScopePathWhen to use
Personal~/.claude/skills/Your workflow, all projects
Project.claude/skills/Team-shared, commit to git
PluginVia claude plugin installOfficial marketplace skills
ℹ️ Skills vs MCP: MCP gives Claude new tools (APIs, databases, file access). Skills give Claude new expertise (how to use those tools, domain standards, review protocols). Use both.

skills/build.md

A skill is a directory with one required file: SKILL.md. Everything else is optional.

File structure

tree
my-skill/ ├── SKILL.md # Required. Frontmatter + instructions. ├── scripts/ # Python/Bash. Executed via bash, only output consumed. │ └── analyze.py ├── references/ # Docs. Loaded on-demand when task references them. │ └── design-tokens.md └── assets/ # Templates, icons, fonts. └── report.docx

SKILL.md structure

yaml + markdown
--- name: design-review description: > Use this skill when asked for a design review, UI critique, accessibility audit, or visual feedback on any component or screen. Triggers on "review this", "how does this look", "accessibility check", or any UI feedback request. --- ## Design Review Protocol When reviewing UI, always check in this order: 1. **Contrast** — WCAG AA minimum (4.5:1 text, 3:1 UI components) 2. **Touch targets** — 44×44px minimum on interactive elements 3. **Spacing** — consistency with token scale (see references/tokens.md) 4. **Motion** — all transitions must respect prefers-reduced-motion 5. **Focus** — visible focus ring on all interactive elements Output format: severity-tagged list - 🔴 CRITICAL — blocks shipping - 🟡 WARN — should fix before release - 🟢 OK — no action needed
⚠️ Description is the trigger. Claude matches tasks to skills using the description field only. Be explicit and slightly over-specify. Undertriggering is the #1 failure mode. Test with 20 prompts — half should trigger, half shouldn't. Aim for >80% precision.

Auto-generate with skill-creator

bash
# Install skill-creator first npx skills add anthropics/skills --skill skill-creator # Then in Claude Code: /skill-creator # Claude will: interview you → draft SKILL.md → run 20 evals # → iterate until trigger accuracy >80% → package for distribution

tools/console-mcp.md ★ 436

Community-built MCP server by southleft. 56+ tools vs 16 in the official remote-only MCP. Use alongside the official one. GitHub ↗ · Docs ↗

Install (NPX — full 56+ tools)

bash
# Get your Figma Personal Access Token from: # figma.com/developers/api#access-tokens (starts with figd_) claude mcp add figma-console -s user \ -e FIGMA_ACCESS_TOKEN=figd_YOUR_TOKEN_HERE \ -e ENABLE_MCP_APPS=true \ -- npx -y figma-console-mcp@latest

Read-only explore (no token needed)

bash
claude mcp add figma-console -s user \ -- npx -y mcp-remote@latest \ https://figma-console-mcp.southleft.com/sse # Note: remote SSE = read-only, 21 tools only # NPX = full write access, 56+ tools, MCP Apps
